搭建ubuntu 工作环境

决定用ubuntu系统炼丹，搭个炼丹的环境

将桌面内容放大

默认的桌面字比较小，由于我是4k的屏，右键桌面然后选择Display setting 把Scale从100% 调到 200%

安装中文输入法

搜索搜狗输入法，进入官网，下载linux版本，会下载一个deb，下到了Downloads里面

在setting里面 找到Region & Language 然后点开 Manage Installed Languages 然后点开Install/Remove Languages 如果Chinese(simplified) 没有勾中 就勾中然后apply 然后选择Keyboard input method system 选fcitx如果没有这个 就sudo apt-get install fcitx 然后点击Apply System-Wide 然后重启电脑

然后通过命令行安装下好的搜过deb

sudo dpkg -i 文件

如果安装过程中缺少依赖就使用sudo apt -f install

删掉firefox 安装 chrome

之前看到一个文章里面是用 dpkg -L ｜ grep firefox 报错 error: —listfiles needs at least one package name argument 换了篇文章 文章链接

使用dpkg —get-selections ｜ grep firefox 就显示出了firefox，firefox-locale-en firefox-locale-zh-hans

之前的那篇文章是用

sudo dpkg -P firefox firefox-locale-en firefox-locale-zh-hans

sudo apt-get purge firefox firefox-locale-en firefox-locale-zh-hans

对于之前的命令 如果改成dpkg -L firefox | grep firefox 有输出，但是都是关于firefox的路径 其中就有/usr/bin/firefox 我在终端里面输入firefox 就能直接打开浏览器

apt-get remove 和 apt-get purge的区别

apt-get remove 会删除软件包而保留软件的配置文件

apt-get purge 会同时清除软件包和软件的配置文件

关于remove和purge案例的链接

就是用remove卸载 手动把配置文件删了，重新install 会出问题，remove之后 dpkg —list 包名 会显示

rc 包名 。。。 r表示remove了 c表示configuration files 还在

最终使用了dpkg -P 来删除包

下载chrome deb安装包

wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb

看到这个google 没有翻墙软件居然也能下，666

下完之后sudo dpkg -i deb文件 在桌面界面点击左下方的9个正方形的图标 然后把chrome 给拖到菜单栏里面去

PS 我在/usr/bin 里面发现 google-chrome 和 google-chrome-stable 随便用哪个都可以打开chrome

我打开github 然后点最大化 结果发现浏览器卡了一会儿 才把页面弄成最大化 郁闷！！

安装nvim

使用vim 发现没有vim 不过会有友好的提示 可以sudo apt install 啥啥啥 而且会告诉install的版本是啥

然后输入vi 结果发现进入的界面居然是vim 有点神奇

PS：发现系统里面有python3 没有python 也没有python2 python3的版本是3.8.10

PS: 发现系统里面有openssl 版本是 1.1.1 31 Mar 2020

听说neovim0.5 是2021年7月份发布的，带来了treesitter和LSP client，0.6是对0.5的一些bug修复

我在github上面的neovim的release中下载nvim.appimage

然后chmod u+x nvim.appimage 然后 ./nvim.appimage 就直接进入nvim的开始界面了

我把这个mv到/usr/bin里面去了，然后sudo ln -s /usr/bin/nvim.appimage /usr/bin/vim

PS 我在这里面发现有nvidia-detector

neovim的官网 neovim.io

安装okular

听说okular是linux下最好的pdf阅读器？

sudo apt-get install okular

进行注释的时候 按下F6 然后再按下数字4就行了 取消注释模式 就是按F6

右键要打开的pdf 选择properties 然后在里面修改默认打开程序

用Ctrl+Shift+，打开Configure Okular 选择Annotations的标签 然后设置Annotation tools

设置不同的highlighter 然后把这些移到最上面 就能方便的使用快捷键来切换使用了

安装一些基本shell软件

sudo apt install htop sudo apt install net-tools 使用htop的时候就能看到和实验室服务器一样的界面了

sudo apt install unzip

安装炼丹套装

  • 安装anaconda 在https://www.anaconda.com/download/#linux 中下载

    上面写着python3.9 64-Bit(x86) Installer 581MB 下载了一个sh文件 下完后bash sh文件

    搞完后的anaconda 里面的base环境 内容贼多，有python3.9.7 我的anaconda3 放在了我的主文件夹下

    然后source ~/.bashrc 郁闷的是base环境内容太多 我之前用conda的时候基本不用base base占了4G空间 有点郁闷

    conda config --show | grep auto_activate_base
会显示出 auto_activate_base: True
使用conda config --set auto_activate_base False 就OK了

  • sudo apt install nvidia-utils-470

    使用nvidia-smi 报错 nvidia-smi has failed because it couldn’t communicate with the nvidia driver

  • 在官网上面搜索GeForce 相应型号显卡的驱动 然后下载

    下载完后 chmod u+x 文件 然后sudo bash 文件

    里面出现一个信息 An alternate method of installing the NVIDIA driver was detected

    我就改用 sudo apt install nvidia-driver-470了

    重启之后 nvidia-smi就ok了 不过cuda version 上面显示的是11.4

    此时nvcc 显示 nvcc not found

  • 在nvidia官网上面搜cuda toolkit 11.1 因为pytorch 官网上面安装pytorch1.8lts 需要的是cudatoolkit 11.1

    然后下载cuda11.1update1 cuda 11.1.1.455.32.00 这个大小是3G

    显示existing package manager installation of the driver found

    继续continue。在installer里面的Driver项 用回车去掉前面的X也就是不安装 因为之前已经安装了 然后install

  • 然后vim ~/.bashrc

    写这篇文章的时候 之前有几天连续头疼 记忆力down down 打vim的时候 忘记后面该打bashrc了

    export PATH="/usr/local/cuda-11.1/bin:$PATH"
export LD_LIBRARY_PATH="/usr/local/cuda-11.1/lib64:$LD_LIBRARY_PATH"
export CUDA_HOME="/usr/local/cuda"

    然后source ~/.bashrc

  • 然后developer.nvidia.com/rdp/cudnn-archive 里面安装相应的cudnn

    下载相应的三个包 我是runtime library for ubuntu20.04 x86_64 (Deb) Developer Library for Ubuntu 20.04 x86_64 (Deb) Code Samples… 下这个要login

    下完之后检测cudnn是否安装成功 cp -r /usr/src/cudnn_samples_v8 到主目录

    然后cd到mnistCUDNN 然后make clean 然后make 可能会报错 FreeImage.h找不到

    sudo apt install libfreeimage3 libfreeimage-dev

    然后./mnistCUDNN 出现Test passed 就成功了

  • 创建conda 环境 我conda create -n myalos python=3.9 发现download的包只有ncurses 说明base里面有的会被我的新环境共享 创建完后使用pytorch 官网提供的命令

    出现conflict了

    我先添加一下清华源

    conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
conda config --set show_channel_urls yes
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/pytorch/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/conda-forge/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/msys2/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/bioconda/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/menpo/

    我把官网给的命令中的-c去掉 结果找不到包 我按官网的命令来 就找到包了 但是安装pytorch 1.8.2的时候非常卡，我以为是因为没有梯子所以我去装梯子了 刚装好 回头一看 pytorch装好了，然后我python

    import torch
torch.cuda.is_available()
# 返回的是True！！！

linux梯子

要下一个客户端和一个核心

我该了端口 socks 10808 http 10809 然后填地址 设置正确的core的路径 就OK了

到此为止我消耗了硬盘空间为42G

连接实验室服务器

直接用终端进行连接

如果要将服务器里面的数据拷贝到我这里的话 用scp -r -P 端口号 用户名@主机ip:文件夹路径 本地文件夹路径

在实验室的服务器上面跑 一个简单的word2vector 需要3个小时 在我原来的dell电脑上是3天，在新电脑上是1个小时

安装oh-my-zsh

首先安装zshell sudo apt install zsh

在oh-my-zsh的主页上有推荐的两种方法

sh -c "$(curl -fsSL https://raw.github.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

要先安装git 我用sudo apt install git 安装的是2.25版的，我在git官网上面看到的是2.34版的

下了一个tarball git 2.34.1.tar.gz 然后

tar -zxf git.tar.gz
cd git-*
make prefix=/usr/local all
sudo make prefix=/usr/local install

报错 make: curl-config: Command not found 安装libcurl4-gnutls-dev

报错 git-compat-util.h:309:10: fatal error: openssl/ssl.h 安装libssl-dev

报错 cache.h:21:10: fatal error: zlib.h: No such file or directory 安装libz-dev

报错 http-push.c:22:10: fatal error: expat.h: No such file or directory 安装libexpat1-dev

报错 * tclsh failed; using unoptimized loading 安装gettext

然后sudo make prefix=/usr/local install

然后执行上面安装oh-my-zsh的命令就OK了

chsh -s /bin/zsh 需要重启才能看到chsh的效果

安装powerlevel 10k

按照github上面的安装教程来，configure font的时候选GNOME Terminal

选择样式的话就按照p10k configure的来就行了

安装autojump 和 zsh-syntax-highlighting 和zsh-syntax-autosuggestions

修改zshrc

plugins=(
	git autojump zsh-autosuggestions zsh-syntax-highlighting
)

PS 装完zshell之后 发现conda不能用了 这个时候将bashrc里面关于conda的内容拷贝到zshrc里面去！！

一套键鼠控制一切

淘宝上面买了Synergy，按照教程操作就行了

安装steam

在steam官网上 下载deb文件

查看cpu温度

sudo apt install lm-sensors hddtemp

sudo sensors-detect 一路yes之后 sensors命令查看温度

k10temp-pci 就是cpu。我的是60多度 可以watch -n 1 sensors来查看动态变化

安装一个硬盘

硬盘里面会附带一根要插在主板上的数据线 和 4个螺丝，有点郁闷的是这4个螺丝不适合我的机箱

换成机箱自带的螺丝 然后装上之后 

sudo mkfs.ext4 /dev/sda
sudo mount /dev/sda 要挂载的目录
sudo chown 用户名 挂载的目录

装完之后还是挺开心的 ～

遇到的问题

输入登录密码之后 黑屏

按Ctrl + alt + f3进入tty3 进行操作

参考了一个博客链接

sudo dpkg --configure -a
sudo apt update
sudo apt upgrade -y
sudo apt install ubuntu-desktop#实际上这条命令没啥反应
sudo apt install lightdm # 更改display manager到lightdm
没有reboot ctrl alt f1到之前的界面还是不行

su密码变了 我把我用户的密码改成原来的su的密码就好了

发现安装的torchvision版本是0.2.2

又发现我的cudnn 好像没有安对， 清华源里面的cudnn的版本是8.0.5 我之前按的是8.2.1 换成8.0.5

用dpkg —list ｜ grep cudnn 发现之前安的包 然后dpkg -P 干掉

下载8.0.5 然后安装 conda install PyTorch torch vision=0.9.2 torchaudio cudatoolkit=11.1 -c PyTorch-lts -c Nvidia

#检查炼丹ok的代码
import pytorch
a = torch.tensor(1.)
a.cuda()
from torch.backends import cudnn
cudnn.is_available()
cudnn.is_acceptable(a.cuda())

ps cudnn的头文件在usr include里面 查看版本查看cudnn version

搞定

然后pip install graphviz 和 tensorwatch

tensorwatch的版本是0.9.1 还要安装sklearn 我手动pip install sklearn 然后conda install ipykernel 还要conda install pandas； 然后用代码

alexnet_model = torchvision.models.alexnet()
